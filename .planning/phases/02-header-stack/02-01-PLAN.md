<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| README.md → readme-typing-svg.demolab.com | Profile visitor's browser fetches an SVG from a third-party demo service; service can change/disappear/return malicious content |
| README.md → img.shields.io | Profile visitor's browser fetches 15 badge SVGs from shields.io; trusted infra but third-party |
| GitHub Markdown sanitizer | Boundary between README source text and what GitHub actually renders; strips `style=`, scripts, untrusted attributes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | readme-typing-svg.demolab.com endpoint | accept | Third-party demo service; if compromised, profile shows wrong/malicious header. Trade-off accepted because Phase 2 is locked to this endpoint by D-05/HEADER-03 and the v2 alternative (capsule-render) was deferred for the same reason. Plan 02 human-verify catches obvious anomalies; long-term mitigation is migration to a self-hosted SVG generator (out of scope). |
| T-02-02 | Denial of Service | readme-typing-svg.demolab.com availability | accept | If demolab.com is down, the header SVG fails to load and the profile shows broken-image alt text "Typing SVG". Non-fatal; rest of profile renders. No mitigation in this phase; alternative endpoints listed in RESEARCH.md §"Alternatives Considered" remain deferred. |
| T-02-03 | Information Disclosure | shields.io badge URLs | accept | Badge URLs are static, committed to a public repo, contain no secrets. No PII, no auth tokens. |
| T-02-04 | Tampering | shields.io badge rendering | accept | shields.io is a long-running, widely trusted service. If a single badge fails to render, it shows the alt text. Acceptable failure mode. |
| T-02-05 | Elevation of Privilege | GitHub Markdown sanitizer bypass | mitigate | We rely on the sanitizer stripping `style=`. Mitigation: explicit acceptance criterion `grep -cE '<[^>]*style=' README.md` returns `0` in BOTH tasks. Any future edit that introduces `style=` is caught by the same grep gate. |
| T-02-06 | Tampering | URL parameter tampering in committed README | mitigate | Static URLs in a committed file mean any tampering would surface in `git diff` review. The `must_haves.key_links` pattern grep (`readme-typing-svg\.demolab\.com.*lines=Data\+Scientist;ML\+Engineer;Full-Stack\+Developer`) acts as a tamper detector during phase verification. |
| T-02-07 | Spoofing | Imposter typing-svg endpoint (e.g. typo squatting `demoIab.com`) | mitigate | Acceptance criterion explicitly greps for the literal string `readme-typing-svg.demolab.com` — homoglyph or typo would not match. |
</threat_model>
